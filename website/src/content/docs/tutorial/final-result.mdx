## Recap

This tutorial moved through Nano Kit in the same order an app tends to grow:

1. Start with one page and one store.
2. Replace manual fetch state with query.
3. Keep inputs reactive and pace expensive search requests.
4. Move shareable state and page matching into the router.
5. Add mutations for page actions and forms.
6. Improve mutations with optimistic updates.
7. Normalize shared records with entities.
8. Prepare stores and router integration for SSR.
9. Add server rendering with `Stores$`, `Head$`, hydration, and a production Hono server.

The important bit is not that every app must use every layer. The useful pattern is adding each layer when the app asks for it.
